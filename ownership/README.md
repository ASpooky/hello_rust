- メモリの主従関係: スタックは高速な処理の主役であり、ヒープは可変長データを置くための外部倉庫にすぎない。

- 他言語の妥協（JS/React）: 「線の共有」によるバグを防ぐため、毎回ヒープを丸写し（Deep Copy）する力技でイミュータブルを作っている。

- Moveの正体: Rustの「所有権の移動」は、ヒープを動かさず、スタック上のポインタ（太い線）だけを付け替える超軽量な操作である。

- コンパイラの魔法: 「借用（細い線）」のルールにより、実行前（ビルド時）にデータの「共有」と「変更」のバッティングを100%防いでいる。

- Rustの真髄: コピーの無駄を省いた『最高速度』のまま、安全に値を書き換えられる『恐れなき可変性』を言語レベルで実現した。
